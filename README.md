# MNIST Digit Recognition — STM32N6570-DK

Real-time handwritten digit classification (0–9) running entirely on the **STM32N6570-DK** Discovery Kit.  
The IMX335 camera captures a live frame, a custom preprocessing pipeline prepares it for the CNN, and the **Neural-ART NPU** runs inference in under 5 ms. The result is printed over UART and shown on the LCD.

## How It Works

### Inference Pipeline

```
IMX335 camera
  └─ DCMIPP pipe 2: downscale → 32×32 RGB888 → nn_raw (uint8[3072])
       │
       │  Pass 1 — Inverted grayscale (Rec.601)
       │    val = (255 − (0.299·R + 0.587·G + 0.114·B)) / 255
       │    → dark ink becomes bright (MNIST convention)
       │    → scan all 1024 pixels to find vmin and vmax
       │
       │  Pass 2 — Adaptive background suppression
       │    midpt = (vmin + vmax) / 2
       │    pixel ≤ midpt  →  0.0          (background zeroed)
       │    pixel > midpt  →  rescaled to [0,1]  (digit preserved)
       │    → eliminates lighting gradients that caused false detections
       │
       ▼
  nn_in (float32[32×32×3] = 12 288 bytes)
       │
       │  Neural-ART NPU — mnist_cnn_32x32_OE_3_3_1
       │  28 394 parameters · weights in NOR flash @ 0x71000000
       │  inference time: < 5 ms @ 1 GHz
       │
       ▼
  nn_out (float32[10]) — softmax probabilities for digits 0–9
       │
       │  argmax · reject if confidence < 70%
       │
       ▼
  LCD overlay + UART  →  [3] 99%  (4ms)
```

See [Doc/preprocessing-and-inference.md](Doc/preprocessing-and-inference.md) for the full technical explanation including why the adaptive suppression was necessary.

## Workflow

### 1 — Train (optional — model already included)

Open `Training/digitsMINST.ipynb` in Google Colab, run all cells. The notebook trains the CNN on MNIST and exports `mnist_cnn_32x32.onnx`. The exported model is then converted for the NPU using STEdgeAI (offline step, output already committed under `Model/`).

### 2 — Build

Open `Application/STM32N6570-DK/STM32CubeIDE/.project` in **STM32CubeIDE v1.17.0**.  
`Project → Build All` → produces `STM32CubeIDE/Debug/Project_sign.bin` (post-build signing is automatic).

### 3 — Flash

Board boot switch in **DEV mode**. From a PowerShell terminal in the project root:

```powershell
.\flash.ps1           # first time: FSBL + app
.\flash.ps1 -AppOnly  # after code changes (faster)
.\flash.ps1 -Weights  # only when the model changes
```

After flashing → switch to **BOOT FROM FLASH** → power cycle.

> Requires [STM32CubeProgrammer v2.18+](https://www.st.com/en/development-tools/stm32cubeprog.html).

### 4 — Monitor

Serial terminal at **115200 baud, 8N1** on USART1 (PE5 TX / PE6 RX).

```
[3] 100%  (4ms)
[4] 96%   (4ms)
[1] 83%   (4ms)
```

## TEST_MODE

Set `#define TEST_MODE 1` in [Application/STM32N6570-DK/Src/main.c](Application/STM32N6570-DK/Src/main.c) to run inference on 10 synthetic images from `digits.h` instead of the live camera. Each image is held for `DIGIT_HOLD_MS` ms and the result is printed with OK/WRONG verdict — useful to verify the model without the physical setup.

## Known Limitations

- Digit **7** handwritten is not recognised — freehand style diverges from MNIST training data.
- Digit **3** handwritten has borderline confidence — use thick strokes and good centering.
- Digits **2↔3** and **1↔7** may confuse when partially out of frame or tilted.
- Background suppression assumes the digit is darker than the background — low contrast scenes produce unreliable output.

---

## Camera Tips

- Hold digit **10–15 cm** from camera, centred in frame
- **Black marker on white paper**, uniform ambient light, no shadows
- Confidence threshold: predictions below **70%** are discarded
