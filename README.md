# MNIST Digit Classification — STM32N6570-DK

Real-time handwritten digit recognition (0–9) on the STM32N6570-DK board.  
The IMX335 camera captures a 32×32 frame, the Neural-ART NPU runs a CNN, and the result is shown on the LCD and printed over UART.

---

## Hardware

| Component | Details |
|-----------|---------|
| Board     | STM32N6570-DK Discovery Kit |
| Camera    | Onboard IMX335 |
| Connection | USB-C → ST-LINK (CN6) |

> **OTP fuses** for xSPI IOs (200 MHz) are programmed automatically on first flash if not already set.

## Workflow

### 1 — Build

Open `Application/STM32N6570-DK/STM32CubeIDE/.project` in **STM32CubeIDE v1.17.0**.  
Build → the IDE produces `STM32CubeIDE/Debug/Project_sign.bin` automatically (post-build signing step included).

### 2 — Flash

Set the board boot switch to **BOOT FROM FLASH**, then from a PowerShell terminal in the project root:

```powershell
# First time (or when FSBL needs updating): flash FSBL + app
.\flash.ps1

# Subsequent builds: app only (faster)
.\flash.ps1 -AppOnly

# When the model changes: include weights
.\flash.ps1 -Weights
```

Power cycle the board after flashing.

> **Requirements:** [STM32CubeProgrammer v2.18+](https://www.st.com/en/development-tools/stm32cubeprog.html) installed in `C:\Program Files\STMicroelectronics\...`

### 3 — Monitor

Connect a serial terminal at **115200 baud, 8N1** on USART1 (PE5 TX / PE6 RX).

Live camera output:
```
[2] 94%  (3ms)
[2] 97%  (3ms)
```

TEST_MODE output (see below):
```
digit=3  predicted=3  conf=91%  OK  (3ms)
digit=7  predicted=7  conf=98%  OK  (3ms)
```

## Inference Pipeline

```
IMX335 camera
  └─ DCMIPP pipe 2: downscale → 32×32 RGB888 → nn_raw (uint8[3072])
       │
       │  Pass 1 — inverted grayscale (Rec.601)
       │    val = (255 − (0.299·R + 0.587·G + 0.114·B)) / 255
       │    → dark ink becomes bright (matches MNIST convention)
       │    → scan all 1024 pixels for vmin, vmax
       │
       │  Pass 2 — adaptive background suppression
       │    midpt = (vmin + vmax) / 2
       │    v = 0                      if val ≤ midpt   (background → 0)
       │    v = (val − midpt) / hi     if val > midpt   (digit → [0,1])
       │    → eliminates lighting gradients that push background to ~0.3
       │
       ▼
  nn_in (float32[32×32×3 = 12 288 bytes])
       │
       │  Neural-ART NPU — mnist_cnn_32x32_OE_3_3_1
       │  Weights in NOR flash @ 0x71000000,  <5 ms @ 1 GHz
       │
       ▼
  nn_out (float32[10]) — softmax probabilities for digits 0–9
       │
       │  Postprocess: rank by probability, reject if top-1 < 70%
       │
       ▼
  LCD overlay + UART
```

## Model

| Property | Value |
|----------|-------|
| File | `mnist_cnn_32x32_OE_3_3_1` |
| Input | 32 × 32 × 3 float32 (RGB, channel-last) |
| Output | 10-class softmax (digits 0–9) |
| Inference time | < 5 ms on Neural-ART NPU @ 1 GHz |
| Weights location | NOR flash @ `0x71000000` |

## TEST_MODE

Set `#define TEST_MODE 1` in [main.c](Application/STM32N6570-DK/Src/main.c) to cycle through 10 synthetic digit images from [digits.h](Application/STM32N6570-DK/Inc/digits.h) instead of using the camera.  
Each digit is held for `DIGIT_HOLD_MS` ms. Use this to verify the model without physical setup, and to capture accuracy logs.

**Capturing a test log (Tera Term):**  
`File → Log…` → choose a file → connect → let it run 2–3 full cycles (≈60 s) → stop.

## Camera Tips

- Hold the digit **10–15 cm** from the camera so it fills most of the frame
- Use **black marker on white paper** under uniform ambient light
- Avoid shadows across the digit
- **Printed digits** (laser/inkjet) classify better than handwriting
- Confidence threshold: predictions below **70%** are rejected

## Known Limitations

- Digits with similar topology (2↔3, 1↔4) may be confused when rotated or partially out of frame.
- The model was trained on MNIST (clean digital-style digits); stylised handwriting reduces accuracy.
- Only RGB888 camera format tested.
